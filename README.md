# 问津 · 知识图谱驱动的个性化学习导航系统

> 大连东软信息学院首届「AI+教育」应用创新大赛 · 赛道一（教育图谱）
> 演示课程：软件工程

**问津**是一个 AI 驱动、任何课程老师都能一键搭建的智能学习导航平台：老师导入课程即可生成知识图谱，学生借助它完成学情诊断、获取个性化学习路径与 AI 学习建议。

可类比「学习版的导航软件」——导航要知道「你在哪、要去哪、怎么走」，并在情况变化时实时重新规划；问津对学习过程做同样的事：**定位学情 → 诊断根因 → 规划路径 → 动态调整**。学生面对一门课「不知卡在哪、不知接下来学什么」，正是身处迷津，而系统所做的一切本质上是一件事——**指点迷津**，故名「问津」。

---

## 一、要解决的三个痛点

1. **学生不知道自己「卡在哪、为什么卡」** —— 做错一道题只知道「这题不会」，却看不到真正薄弱的前置知识点。
2. **学习缺乏个性化路径** —— 所有人按同一套教材同一顺序学，无法按个人掌握情况动态调整「接下来该学什么」。
3. **知识图谱构建门槛高** —— 个性化学习的前提是知识图谱，而梳理知识点依赖关系是高度耗费人力的工作，导致同类系统大多停留在单门写死的课程上。

其中第 3 点是赛题公认最难的环节，问津通过 **「AI 辅助建图谱 + 标准化导入」** 把这一难点转化为产品亮点。

## 二、系统架构（四层，数据单向流动并形成闭环）

```
┌─────────────────────────────────────────────┐
│  ① 图谱构建层（教师端）                          │
│     AI 自动建图谱 / 结构化导入 / 可视化微调        │
└───────────────────┬─────────────────────────┘
                    │ 统一数据契约（点 + 边 JSON）
┌───────────────────▼─────────────────────────┐
│  ② 数据与存储层                                 │
│     知识图谱（节点表 + 边表）/ 多课程隔离 / 学情数据 │
└───────────────────┬─────────────────────────┘
                    │
┌───────────────────▼─────────────────────────┐
│  ③ 智能核心层                                   │
│     学生画像 / 认知诊断 / 动态路径 / AI 学习伴侣    │
└───────────────────┬─────────────────────────┘
                    │
┌───────────────────▼─────────────────────────┐
│  ④ 体验呈现层（学生端）                          │
│     知识地图 / 学习路径 / 成长档案 / 资源匹配       │
└─────────────────────────────────────────────┘
```

**统一数据契约**：无论图谱来自 AI 自动生成、结构化文件导入还是手动编辑，最终都收敛为同一套内部结构（节点 + 边）。三种构建方式殊途同归，是架构的核心设计之一。系统从设计之初即支持多课程（`course_id` 多租户隔离），不为后续扩展返工。

## 三、智能核心层 —— 项目的灵魂（最终形态）

按「学生使用流程」组织为四个能力：

### 🧭 学生画像（定位：你在哪）
- 进入后完成一套覆盖课程主要知识点的**入口诊断小测**（客观题为主，**20–30 题、约 15–20 分钟**）。
- 组卷采用**分层抽样**（按章节分配题量、覆盖枢纽点与难度代表点）+ **图结构推断**（沿前置链按衰减传播掌握度，一份小测的信息量被图谱放大到全图）。
- 为每个知识点计算**掌握度**并分级（已掌握 / 薄弱 / 未学），映射到图谱形成「**染色地图**」，一眼看清薄弱区域。
- 掌握度用 **EWMA 近期加权更新**（`新掌握度 = α × 本次表现 + (1−α) × 旧掌握度`，默认 α=0.3），保证新表现能覆盖旧表现——连续答对几题，地图就能从「薄弱」变绿回「已掌握」，动态自适应不再是假动态。

### 🔍 认知诊断（诊断：你为什么卡）
- 不止于「对错打分」，而是**反向定位薄弱根因**：答错某点时顺着图谱**前置边逆向回溯（最多 2 层）**，检查前置点掌握情况。
- 主 / 次权重参与判定；前置点无数据则推送 1–2 道验证题「用数据说话」；多个前置薄弱按「权重 × 掌握度缺口」给出嫌疑列表。
- **结论可解释、可追问**：如「你 UML 图掌握不好，根本原因是需求建模基础未吃透（前置点掌握度仅 35），建议先回补」。

### 🛤️ 动态自适应路径（导航：接下来学什么）
- 在图上做**拓扑排序**（学一个知识点前其前置必须先学），生成有序学习路径。
- **动态自适应**：每次学习 / 答题后实时重算剩余路径——已掌握的不再推荐，把精力导向仍薄弱处；路径每一步关联对应学习资源。

### 🤖 AI 学习伴侣（陪伴：随时可问）
- 接入大模型，结合学生当前画像与其在图谱中的位置进行**对话式答疑**，能回答「为什么我要先学需求分析才能学 UML 建模」这类与个人学情绑定的问题。
- **防幻觉护栏**：system prompt 注入本课程知识点白名单与学生画像，回答**限定在本课程图谱范围内**，超纲问题礼貌引导回主题。

## 四、教师端图谱构建（三种方式，殊途同归）

1. **AI 自动建图谱（核心亮点）** —— AI 生成知识点与关系初稿，人工修正定稿；价值不在「存两张表」，而在解决「知识点之间到底什么关系」这一最难的人工环节，并以节点召回率 / 边准确率量化呈现。
2. **结构化文件导入** —— 按标准模板（Excel/JSON）批量导入节点与边，导入时做严格校验（节点 ID 唯一、边端点存在、关系类型合法、**前置边环检测**）。
3. **可视化微调** —— 在图上拖拽增删节点 / 边，所见即所得。

## 九、快速开始

### 0. 环境要求

| 组件 | 版本 | 说明 |
| --- | --- | --- |
| JDK | 21 | 后端 |
| Maven | 3.6.3+ | 仓库内置 **Maven Wrapper（`mvnw`，固定 3.9.9）**，无需本机装 Maven |
| MySQL | 8.0+ | utf8mb4 |
| Node.js | 18+ / 20+ | 前端构建 |

### 1. 建库

```bash
cd wenjin-server
mysql -u root -p < src/main/resources/schema.sql
# 既有库做增量迁移（已建过库时执行，保留数据）：
mysql -u root -p wenjin < seed/migration_phase2.sql
```

`schema.sql` 建全 14 张表（带中文注释、可重复执行），并种入演示教师、演示课程（`course.id=1`, `code=52015CC4B4`）与演示学生（`id=2`）。

### 2. 配置（数据库 / AI）

后端默认连 `localhost:3306/wenjin`，账号 `root` / 密码 `root`，可改 `application.yml` 或用环境变量 `DB_USER` / `DB_PASSWORD` 覆盖。

AI 出题 / 标注需要可插拔大模型的 Key（OpenAI 兼容，默认 DeepSeek）：

```powershell
$env:WENJIN_AI_API_KEY = "你的 Key"
# 可选：$env:WENJIN_AI_BASE_URL / $env:WENJIN_AI_MODEL 覆盖默认 DeepSeek
```

### 3. 启动后端（端口 8080）

```bash
cd wenjin-server
mvnw.cmd spring-boot:run     # Windows
./mvnw spring-boot:run       # macOS / Linux
```

### 4. 导入种子数据

```bash
# 图谱（同课程重复导入 = 全量替换）
curl -X POST "http://localhost:8080/api/admin/graph/import?courseCode=52015CC4B4" \
     -H "Content-Type: application/json" \
     --data-binary "@seed/问津_软件工程图谱_v0.3.json"

# 入口诊断种子题库（25 题，status=已通过）
curl -X POST "http://localhost:8080/api/admin/question/import-bank?courseCode=52015CC4B4"
```

### 5. 启动前端（端口 5173）

```bash
cd wenjin-web
npm install
npm run dev
```

浏览器打开 **http://localhost:5173**（开发服务器把 `/api` 代理到后端 8080）。

## 十、接口一览（已实现）

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/admin/graph/import?courseCode=` | 导入图谱（校验 + 全量替换） |
| GET | `/api/graph/{courseId}` | 查询课程全部节点与边（含掌握度，用于染色地图） |
| POST | `/api/admin/question/generate?nodeCode=&count=` | AI 出题即标注（白名单内、代码侧校验、落待审核） |
| POST | `/api/admin/question/annotate` | 存量题 AI 标注（全图白名单，超纲不强标） |
| POST | `/api/admin/question/import-bank?courseCode=` | 导入种子题库（落已通过） |

> 入口诊断组卷 `GET /api/diagnostic/paper`、交卷判分 `POST /api/diagnostic/submit` 等学生端接口开发中。
> 统一返回体 `Result<T>`：`{ code, message, data }`，`code=0` 为成功。
